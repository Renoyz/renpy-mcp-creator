# 项目背景 Brief — 给设计 Agent

## 1. 项目定位
**"会说话就能做 Galgame"**

这是一款面向中国创作者的一站式 AI 视觉小说生成器。用户不需要懂代码、不需要会画画，只需要用中文描述故事，AI 就能产出带立绘、背景、分支剧情的可玩 Web 游戏。

## 2. 核心用户
- **主用户**：18-30 岁的视觉小说创作者，有故事创意但不懂编程
- **次要用户**：懂一点 Ren'Py 的独立开发者，想快速出 Demo

## 3. 核心技术架构
### Spec 模式（先规划后生成）
用户创作流程分为 6 个阶段：
1. 用户用自然语言描述需求
2. AI 通过提问收敛需求
3. 生成并展示 `ProjectBlueprint`（项目蓝图/大纲）
4. 用户确认 Blueprint
5. AI 按 Chapter → Scene 的粒度逐段生成脚本和资源
6. 每完成一章自动审计并构建试玩

### 双 Agent 架构
- **Creator Agent**：负责生成脚本、生成图片资源、写代码（ creative 角色）
- **Auditor Agent**：负责检查一致性、审计质量、发现错误（ reviewer 角色）
- 两个 Agent 通过 Orchestrator 调度，形成"生成 → 审计 → 修复"的闭环

## 4. 当前 UI 的核心问题
当前 Dashboard 是 IDE 式布局：左侧固定导航（项目、Story Map、脚本编辑、资源管理），右侧主内容区。这种设计与新的"AI 生成 + 蓝图确认 + 进度审计" workflow 严重不匹配。

**关键问题**：
- 没有 Blueprint 展示入口
- 没有创作进度可视化
- Chat（AI 对话）被当作可选辅助功能放在顶部栏，但它是创作主入口
- Story Map / 脚本编辑 / 资源管理用 iframe 割裂，无法联动
- 审计结果没有专属展示页面

## 5. 设计目标
需要一套新的 UI 设计，将 Dashboard 从"工具箱"转变为"创作进度驾驶舱"：
- **Blueprint（项目蓝图）**必须成为项目的核心视图
- **Chapter/Scene 时间线**必须成为主导航单位
- **Chat（AI 对话）**必须得到与其重要性匹配的位置和呈现方式
- **Audit Report（审计报告）**必须有独立的审阅空间
- 支持从 Story Map 节点直接跳转到对应 Scene 的脚本编辑

## 6. 关键概念
| 术语 | 说明 |
|------|------|
| **ProjectBlueprint** | 项目大纲，包含角色表(Cast)、章节表(Chapters/Scenes)、资源清单、叙事基调 |
| **Chapter** | 故事章节，由 2-4 个 Scene 组成。每章生成完后自动审计+试玩 |
| **Scene** | 最小生成单元，对应一个 Ren'Py `label` 块，包含台词、背景、角色情绪 |
| **Audit Report** | 结构化审计报告，检查跨 Scene 一致性、Blueprint 对齐、资源覆盖、叙事基调、可玩性 |
| **Chat / AI 助手** | 统一对话引擎的前端呈现，创作者通过自然语言与 AI 交互的主入口 |

## 8. 设计约束
- 使用深色/浅色双主题兼容的配色系统
- 优先考虑中文排版（文字信息密度高于英文）
- 保持简洁，避免过度装饰
- 创作者通常连续使用 30 分钟以上，需要考虑视觉疲劳
